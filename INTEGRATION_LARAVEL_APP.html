⚠️
Avant d'écrire ce controller

Vérifiez comment vos numéros de magazine sont stockés dans votre base de données. Cherchez le nom de la table dans votre fichier database/migrations/ ou dans PhpMyAdmin/TablePlus. Adaptez le nom de table dans le code ci-dessous.

app/Http/Controllers/Api/MagazineController.php → créer 
<?php

namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use Illuminate\Support\Facades\DB;

class MagazineController extends Controller
{
    public function index()
    {
        // ── Adapter le nom de table selon votre base de données ──────────
        // Exemples courants : 'magazines', 'issues', 'revues', 'editions'
        $issues = DB::table('magazines')        // ← changer si besoin
            ->where('status', 'published')      // ← adapter le champ statut
            ->orderBy('number', 'desc')
            ->get([
                'id',
                'number',      // numéro : 21, 20, 19...
                'label',       // ex: "N°21 · Oct. 2025"
                'theme',       // titre/thème du numéro
                'free',        // gratuit : true/false
                'price',       // ex: "1 250 FCFA"
                'cover_url',   // URL de la couverture (image)
            ]);

        return response()->json(['data' => $issues]);
    }
}
ℹ️
Si vos colonnes ont des noms différents

Utilisez des alias SQL : DB::table('magazines')->select('id', 'numero as number', 'titre as theme', ...). L'app attend exactement ces noms : id, number, label, theme, free, price, cover_url.

4 App mobile — code à modifier

src/services/api.ts — ajouter à la fin du fichier existant

src/services/api.ts → ajouter à la fin 
// ─── Recherche ────────────────────────────────────────────────────────────────

export async function searchArticles(query: string): Promise<ApiArticle[]> {
  if (!query.trim()) return [];
  try {
    const res = await fetch(
      `${BASE}/articles?search=${encodeURIComponent(query)}&per_page=20`,
      { headers: { Accept: 'application/json' } }
    );
    if (!res.ok) return [];
    const json = await res.json() as PaginatedResponse<ApiArticle>;
    return json.data ?? [];
  } catch {
    return [];
  }
}

// ─── Magazine ────────────────────────────────────────────────────────────────

export interface ApiMagazineIssue {
  id: number;
  number: number;
  label: string;
  theme: string;
  free: boolean;
  price: string;
  cover_url: string | null;
}

export async function fetchMagazineIssues(): Promise<ApiMagazineIssue[]> {
  const res = await apiFetch<{ data: ApiMagazineIssue[] }>(
    '/magazine/issues', 'magazine_issues'
  );
  return res?.data ?? [];
}

// ─── Abonnements ─────────────────────────────────────────────────────────────

export interface ApiPlan {
  id: string;
  name: string;
  price_year: string;
  price_month: string;
  description: string;
  features: string[];
  recommended?: boolean;
}

export async function fetchSubscriptionPlans(): Promise<ApiPlan[]> {
  const res = await apiFetch<{ data: ApiPlan[] }>(
    '/subscriptions/plans', 'subscription_plans', 30 * 60 * 1000 // cache 30 min
  );
  return res?.data ?? [];
}
⚠️
Modifier apiFetch pour accepter un TTL optionnel

La fonction apiFetch dans api.ts doit accepter un 3e paramètre ttl. Modifiez sa signature : async function apiFetch<T>(path: string, cacheKey?: string, ttl = CACHE_TTL) et passez ce ttl à cacheGet.

6 Vidéos YouTube

L'endpoint GET /api/videos existe déjà et retourne 3 vidéos. Côté Laravel : rien à faire. Les modifications suivantes ont été apportées à l'app pour afficher ces vidéos.

✅
Endpoint déjà opérationnel

Testé en direct : GET /api/videos retourne 3 vidéos avec youtube_id, title et thumbnail_url. Aucun code Laravel à toucher.

src/services/api.ts — ajouté

La fonction fetchVideos() et l'interface ApiVideo ont été ajoutées dans le fichier.

src/services/api.ts→ ajouté
export interface ApiVideo {
  id: number;
  title: string;
  youtube_id: string;
  thumbnail_url: string | null;
  published_at: string | null;
}

export async function fetchVideos(): Promise<ApiVideo[]> {
  try {
    const res = await fetch(`${BASE}/videos`, {
      headers: { Accept: 'application/json' },
    });
    if (!res.ok) return [];
    const json = await res.json() as { items: ApiVideo[] };
    return json.items ?? [];
  } catch {
    return [];
  }
}

Un composant VideoSection a été créé et inséré dans le feed de l'accueil, après la section "Plus lus". Il affiche un scroll horizontal de cartes vidéo avec miniature YouTube et bouton play. Un tap ouvre la vidéo dans YouTube.

// Composant VideoSection — scroll horizontal de miniatures YouTube
const VideoSection: React.FC<{ videos: ApiVideo[] }> = ({ videos }) => {
  if (videos.length === 0) return null;
  return (
    <View style={{ marginTop: Spacing["4"] }}>
      <Text style={{ /* titre "Nos vidéos" */ }}>Nos vidéos</Text>
      <ScrollView horizontal showsHorizontalScrollIndicator={false}>
        {videos.map((v) => {
          const thumb = `https://img.youtube.com/vi/${v.youtube_id}/hqdefault.jpg`;
          const url   = `https://www.youtube.com/watch?v=${v.youtube_id}`;
          return (
            <TouchableOpacity onPress={() => Linking.openURL(url)}>
              <Image source={{ uri: thumb }} />
              <View style={{ /* bouton play blanc centré */ }}>
                <Feather name="play" size={16} color="#1C1C1E" />
              </View>
              <Text numberOfLines={2}>{v.title}</Text>
            </TouchableOpacity>
          );
        })}
      </ScrollView>
    </View>
  );
};

const [videos, setVideos] = useState<ApiVideo[]>([]);
